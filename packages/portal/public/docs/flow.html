包结构

流程引擎分为三个子包：shared（共享类型与引擎工具）、server（后端服务）、web（前端设计器）。

@schema-form/flow-shared

共享层 — BPMN 类型定义、图数据结构、可执行模型、表达式求值器、BPMN XML 导入/导出。无运行时依赖。

types/ ExecutableModel BpmnParser ExpressionEvaluator BpmnXmlExporter BpmnXmlImporter

@schema-form/flow-server

后端层 — Mongoose 数据模型、Koa 路由、Token 驱动执行引擎、任务服务、定时器服务、权限服务。

FlowEngine TaskService TimerService PermissionService

@schema-form/flow-web

前端层 — Vue Flow 画布、12 种 BPMN 节点组件、属性面板、工具栏、用户选择器、Pinia Store。

FlowDesigner FlowCanvas FlowPalette FlowPropertyPanel

Token 执行模型

流程引擎采用 Token 驱动的执行模型，Token 在节点间流转，驱动流程推进。

StartEvent → UserTask → ExclusiveGateway → ServiceTask → EndEvent

Token 生命周期

Token 在 StartEvent 创建，沿边流转到各节点。到达 UserTask 时挂起等待审批，审批完成后继续推进。并行网关时 Token 分叉，包容网关时 Token 合并。

advance() 循环

引擎核心是 advance() 循环：取一个活跃 Token，根据当前节点类型执行对应 handler，将 Token 推进到下一个节点，直到流程结束或等待外部事件。

审批流程

支持单人审批、会签（全部通过）、或签（任一通过）三种模式。

👤 单人审批

指定唯一审批人，审批完成后流程继续。

approve reject

✍️ 会签（Countersign）

多人同时审批，全部通过才通过。支持驳回到指定节点。

全员通过 可驳回

✋ 或签（Or-Sign）

多人审批，任一通过即通过。未审批的任务自动取消。

任一通过 自动取消

数据模型

FlowDefinition

流程定义。包含 name、description、formSchemaId（关联表单）、bpmnGraph（Vue Flow 图数据）、status（draft/published）、permissions。

FlowVersion

流程版本快照。每次发布创建新版本，记录 bpmnGraph、nodeConfigs、approvalMode、rejectPolicy 等完整配置。

FlowInstance

流程实例。运行中的流程实例，包含 tokens（Token 集合）、variables（流程变量）、status（running/completed/terminated/suspended）。

TaskInstance

任务实例。用户审批任务，包含 assignee（审批人）、status（pending/completed/delegated）、approvalMode、formData。

ApprovalLog

审批日志。记录每次审批操作：taskId、instanceId、operator、action、comment、timestamp。

TimerJob

定时任务。ISO 8601 间隔配置，Cron 触发器驱动定时事件评估，用于超时提醒等场景。

权限体系

流程级权限

FlowPermissionService 控制流程级别的访问权限：谁能启动流程、谁能查看实例、谁能管理流程定义。

节点级权限

每个 UserTask 节点可独立配置审批人来源：指定用户、角色、部门、变量表达式动态计算。
