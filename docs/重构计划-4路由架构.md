**日期**: 2026-05-16

## 一、目标

将现有6个松散路由重构为4个清晰路由，打通编辑器→服务端→渲染器完整链路。

| 新路由 | 名称 | 说明 |
|--------|------|------|
| `/instances` | 实例管理 | 卡片列表，创建/编辑/预览/发布/删除实例 |
| `/editor?id={uuid}` | 编辑器 | 可视化编辑表单和搜索列表 |
| `/view?id={uuid}` | 发布预览 | 公开渲染器，仅展示已发布的实例 |
| `/docs` + `/docs/:componentId` | 组件文档 | 每个组件独立文档页 |

**核心原则**: 
- 一个实例 = 一个 UUID = 编辑和发布的唯一标识
- 发布不创建新ID，只改变status
- 编辑器退出前必须提示保存
- 所有数据走 Pinia stores + composables，不走本地ref
- Mock数据仅属于组件文档，不属于生产组件
