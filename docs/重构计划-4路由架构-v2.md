## 核心概念

### 两条数据线

```
编辑线（edit ID = 实例 UUID）
  │
  ├─ 创建 → POST /api/schemas → 获得 UUID
  ├─ 编辑 → /editor?id={UUID} → GET → 加载 json
  ├─ 保存 → PUT {json, status:'draft'}
  ├─ 预览 → /preview?id={UUID} → 渲染当前编辑数据
  │
  └─ 发布 → PUT {json, status:'published'}
                │
                ▼
发布线（publish ID = 发布时生成的唯一 UUID）
  │
  └─ 查看 → /view?id={publish_id} → 渲染已发布版本
      首次发布生成 publish_id，后续发布复用同一个
```

**关键规则**:
- 编辑 ID ≠ 发布 ID（两个不同的 UUID）
- 发布 ID 首次发布时生成，后续发布复用同一个
- /preview 看编辑数据（草稿），/view 看发布数据（已发布）
- 编辑器未携带 id 时不请求接口，直接展示空白编辑器
- 保存后自动获得 id，拼入 URL，后续操作走服务

## 一、DB Schema

```prisma
model FormSchema {
  id           String    @id @default(uuid()) @db.Uuid
  name         String
  type         String    @default("form")      // 'form' | 'search-list'
  status       String    @default("draft")     // 'draft' | 'published'
  publishId    String?   @unique @db.Uuid      // 发布时生成，null=从未发布
  publishedAt  DateTime?                       // 最后发布时间
  json         Json                            // 当前编辑数据（非发布快照）
  createdAt    DateTime  @default(now()) @map("created_at")
  updatedAt    DateTime  @updatedAt @map("updated_at")

  @@map("form_schemas")
}
```

### 发布时服务端逻辑

```typescript
// PUT /api/schemas/:id 中 status='published' 时:
if (status === 'published') {
  if (!existing.publishId) {
    data.publishId = uuidv4()  // 首次发布：生成 publish_id
  }
  // 后续发布：publishId 不变
  data.publishedAt = new Date()
}
```

## 二、路由设计

| 路由 | 视图 | Query | 说明 |
|------|------|-------|------|
| `/instances` | InstancesView | — | 卡片网格，创建/管理实例 |

```
/            → /instances
/schemas     → /instances
/renderer    → /view
/demo        → /docs
/preview     → /docs  (组件文档)
```

## 四、数据流

### 4.1 创建实例

```
用户点击 [新建表单] 或 [新建搜索列表]
  → ElMessageBox.prompt("实例名称")
  → POST /api/schemas { name, type, json: [] }
  → 获得 { id: "uuid-xxx" }
  → router.push('/editor?id=uuid-xxx')
```

### 4.2 编辑实例

```
/editor?id=uuid-xxx
  → onMounted 检测 route.query.id
  → 有 id: GET /api/schemas/uuid-xxx → schema.value = data.json
  → 无 id: schema.value = []（空白画布）
```

### 4.3 保存草稿

```
点击 [保存草稿]
  → 无 id: POST /api/schemas { name, type, json, status:'draft' }
       → router.replace('/editor?id={新id}')
  → 有 id: PUT /api/schemas/:id { json, status:'draft' }
       → 重新 GET /api/schemas/:id 刷新编辑器数据
```

### 4.4 发布

```
点击 [发布]
  → PUT /api/schemas/:id { json, status:'published' }
  → 服务端: 首次发布生成 publishId + publishedAt，后续发布复用 publishId
  → 返回 { publishId: "pub-uuid-xxx" }
  → 提示 "发布成功"，可跳转 /view?id=pub-uuid-xxx
```

### 4.5 编辑器预览

```
/preview?id=uuid-xxx（编辑ID）
  → GET /api/schemas/uuid-xxx
  → 渲染当前编辑数据（不需要status='published'）
  → 显示 "预览模式 - 草稿" 横幅
```

### 4.6 发布查看

```
/view?id=pub-uuid-xxx（发布ID）
  → GET /api/schemas?publishId=pub-uuid-xxx（或专门端点）
  → 仅 status='published' 可访问
  → 完整渲染
```

### 4.7 退出保护

```
编辑器 beforeRouteLeave:
  → 对比当前 json === lastSavedJson
  → 无变化: 直接离开
  → 有变化: ElMessageBox "有未保存更改" [保存] [不保存] [取消]
```

## 五、API 变更

### 5.1 GET /api/schemas（列表）

新增查询参数: `type`, `status`
响应新增字段: `type`, `status`, `publishId`, `publishedAt`

### 5.2 POST /api/schemas（创建）

请求体新增: `type`(必填)
响应: 完整 SchemaDetail

### 5.3 PUT /api/schemas/:id（更新）

请求体新增: `status`
服务端逻辑: status='published' 时自动处理 publishId + publishedAt

### 5.4 GET /api/schemas?publishId=xxx（按发布ID查询）

新增查询方式，供 /view 页面使用
