8. 界面风格一致性分析与重构方案

🚨 核心问题

各项目样式定义分散、变量命名不统一、主题覆盖方式各异，导致视觉风格不一致。

📊 现状分析
项目样式入口主色变量主题覆盖方式问题
Shell tokens.css --color-primary: #0060A2 CSS 变量 + Element Plus 默认 OK
Admin tokens.css --color-primary: #0060A2 CSS 变量 + Element Plus 默认 OK
AI App tokens.css --color-primary: #0060A2 CSS 变量 + Element Plus 默认 OK
Editor tokens.css + styles/theme.scss + styles/variables.scss $fg-color-primary: #0060A2
$color-primary: #0060A2		 SCSS 变量 → Element Plus CSS 变量覆盖 重复定义
Flow tokens.css + styles/theme.scss（引用 shared） $color-primary: #0060A2 引用 shared/_theme → Element Plus 覆盖 两层覆盖
🔍 具体问题
#问题影响
1 Editor 有独立的 variables.scss（$fg-color-primary）与 shared 的 _variables.scss（$color-primary）重复 改一处忘改另一处，颜色不一致
2 Editor 的 theme.scss 和 shared 的 _theme.scss 都覆盖 Element Plus 变量 两层覆盖，优先级混乱
3 Shell 侧边栏深色（#001529），Admin 侧边栏浅色（白色） 切换应用时视觉跳脱
4 Flow 的 theme.scss 用 @use '../../../../shared/styles/theme' 相对路径 路径脆弱，移动目录即报错
5 component-styles.css 定义了 .sf-button 等类，但各项目未统一使用 公共样式形同虚设
🎯 重构方案：统一到 TDesign 设计体系

参考资源：

  • Figma 设计稿：TDesign for Web Community
  • 组件库：TDesign Vue Next

Phase 1：统一设计令牌（shared/styles）

packages/shared/styles/
├── tokens.css              # CSS 变量（所有项目引用）
│   ├── --td-brand-color    # TDesign 主色
│   ├── --td-brand-color-hover
│   ├── --td-brand-color-light
│   ├── --td-font-family
│   ├── --td-font-size-*
│   ├── --td-spacing-*
│   ├── --td-radius-*
│   └── ...
├── element-plus-override.scss  # Element Plus → TDesign 变量映射
└── index.css               # 统一入口

Phase 2：删除各项目重复定义

  • 删除 editor/web/src/styles/variables.scss（合并到 shared）
  • 删除 editor/web/src/styles/theme.scss（统一到 shared）
  • 简化 flow/web/src/styles/theme.scss（只保留 flow 特有覆盖）
  • 统一侧边栏风格（Shell + Admin 都用深色或都用浅色）

Phase 3：引入 TDesign 组件（可选）

评估 Element Plus → TDesign Vue 的迁移成本，或仅采用 TDesign 的设计令牌映射到 Element Plus。
