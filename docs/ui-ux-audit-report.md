# UI/UX 审查报告

## 一、对比 Dify/N8n 的差距

### 1.1 与 Dify 的差距

| 维度 | Dify | 我们 | 差距 | 改进方案 |
|------|------|------|------|----------|
| 变量自动补全 | 输入框内自动触发补全 | VariableSelector 独立 popover | 体验差距大 | 将 VariableSelector 集成到输入框内，输入 `{{` 时自动触发 |
| 模板市场 | 丰富的社区模板库 | WidgetTemplateView 有基础模板 | 内容差距 | 建立模板导入/分享机制，丰富预置模板 |
| API 文档 | 发布后自动生成 REST API | 无 | 功能缺失 | Schema 发布后自动生成 API 端点文档 |
| 知识库 | 完整的 RAG 知识库管理 | AiRagSearch 组件存在 | 功能深度差距 | 完善知识库管理界面 |
| 工作流编排 | 自由连线 + 节点卡片 | BPMN 标准设计器 | 风格差异 | 我们更专业，保持 BPMN 标准 |

### 1.2 与 N8n 的差距

| 维度 | N8n | 我们 | 差距 | 改进方案 |
|------|------|------|------|----------|
| 节点运行状态 | 实时状态指示器（绿勾/红叉/旋转） | FlowMonitorDashboard 有统计 | 缺少节点级实时状态 | 在流程实例详情中高亮当前执行节点 |
| 执行日志 | 节点级输入/输出/耗时日志 | 有基础日志 | 粒度不足 | 添加节点级执行日志面板 |
| 调试模式 | 可单步执行、查看中间数据 | 无 | 功能缺失 | 未来考虑添加流程调试模式 |
| 凭证管理 | 独立模块 + 加密存储 | Credential 模型已实现 | 已补齐 | ✅ 完成 |

### 1.3 我们的独特优势

| 维度 | 我们 | Dify/N8n |
|------|------|----------|
| 表单设计器 | 49 个 Widget，自由布局拖拽 | Dify 无表单设计器，N8n 无 |
| BPMN 标准 | 完整 BPMN 2.0 节点支持 | N8n 是自定义节点模型 |
| 多租户 RBAC | 完整的租户/角色/权限体系 | Dify 有基础 RBAC，N8n 无 |
| SSO 单点登录 | OAuth2 标准 SSO | 需要企业版 |
| 微前端集成 | qiankun + micro-app 双模式 | 无 |
| 审批流程 | 完整的审批/驳回/委派/批量 | Dify 无审批，N8n 无 |

## 四、改进建议

### 4.1 短期改进（1-2 周）

1. **EditorLeftPanel CSS Module 迁移** — 修复架构违规
2. **统一错误边界** — ErrorBoundary 抽取为共享组件

1. **变量自动补全集成** — 输入 `{{` 时自动触发 VariableSelector
2. **节点级执行日志** — 流程实例详情中添加执行日志标签页
3. **模板市场** — 建立模板导入/分享机制
4. **API 文档自动生成** — Schema 发布后自动生成 API 端点文档

### 4.3 长期改进（3-6 月）

1. **流程调试模式** — 单步执行、查看中间数据
2. **深色模式** — Element Plus 原生支持暗色主题
3. **响应式适配** — 编辑器和流程设计器支持平板布局
4. **知识库管理** — 完善 RAG 知识库管理界面
