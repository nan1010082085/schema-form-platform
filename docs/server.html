中间件栈

请求按顺序经过以下中间件处理。

1
errorHandler
统一错误捕获，返回标准化错误响应
2
helmet
安全 HTTP 头（XSS、Content-Security-Policy 等）
3
bodyParser
JSON 请求体解析
4
CORS
跨域资源共享，通过 CORS_ORIGINS 环境变量控制
5
auth (JWT)
JWT 令牌验证，bcryptjs 密码哈希
6
rateLimit
请求频率限制（100 req/15min/IP）
7
validate (Zod)
请求参数 Schema 校验

数据模型

FormSchema

核心资源。UUID 主键，name、type（form/search_list）、status（draft/published）、json（Widget[] 树结构）、publishId、时间戳。

PublishedSchema

发布版本快照。关联 sourceId，记录每次发布的完整 Schema 快照。

User

用户账户。username、password（bcrypt 哈希）、JWT 令牌签发。

FlowDefinition

流程定义。bpmnGraph、nodeConfigs、formSchemaId、permissions、status。

FlowInstance

流程实例。tokens、variables、status（running/completed/terminated/suspended）。

TaskInstance

任务实例。assignee、approvalMode、status（pending/completed/delegated）、formData。

Vercel 部署

部署于 Vercel Serverless，512MB 内存，30s 最大执行时间。

Serverless 入口

api/index.ts → packages/server/dist/handler.js。MongoDB 连接惰性初始化，支持重连和 503 降级。

Cron 任务

每分钟触发 /api/flow-timers/check，评估流程定时器，推动超时任务自动处理。

路由规则

/api/* → Serverless Function。其他路径 → SPA（index.html）。静态资源 1 年缓存。

构建流程

pnpm build:server（tsc 编译）→ pnpm build:editor（Vite 构建）。前后端独立构建。

环境变量

变量用途必需
MONGODB_URIMongoDB 连接字符串必需
NODE_ENVdevelopment / production可选
PORT本地开发端口（默认 3001）可选
CORS_ORIGINS允许的跨域来源，逗号分隔可选

快速开始

# 启动本地 MongoDB
pnpm db:up

# 填充种子数据
pnpm db:seed

# 启动后端开发服务器（端口 3001，tsx watch 热重载）
pnpm dev:server

# 构建后端 TypeScript
pnpm build:server

# 健康检查
curl http://localhost:3001/api/health
