## 二、业务闭环架构

### 2.1 核心业务流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        业务闭环：Editor → Flow → 审批                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  阶段 1：表单设计（Editor）                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  设计师在 Editor 中拖拽配置表单 Schema                            │   │
│  │  发布表单获得 publishId                                          │   │
│  │  状态：✅ 已完整实现                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  阶段 2：流程设计（Flow）                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  流程设计师在 Flow 中配置 BPMN 流程                              │   │
│  │  每个 UserTask 节点绑定已发布的表单（formSchemaId + formPublishId）│   │
│  │  配置审批人（user/role/expression）、审批模式                     │   │
│  │  状态：✅ 已完整实现                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  阶段 3：业务界面（Editor）                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Editor 设计业务列表页/表单页                                     │   │
│  │  按钮配置：启动流程 / 结束流程 / 自定义 API                       │   │
│  │  弹框表单：嵌入渲染器，支持微应用或界面内嵌入                     │   │
│  │  状态：🆕 需要增强按钮事件系统                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  阶段 4：审批处理（Flow）                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  审批人在 TaskInboxView 看到待办任务                              │   │
│  │  签收 → 查看/编辑绑定的 Editor 表单 → 提交 → 完成任务            │   │
│  │  审批表单模式：完全只读 / 部分可编辑                              │   │
│  │  状态：🆕 需要增强审批表单模式                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  阶段 5：流程结束（Flow）                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  所有节点执行完毕，FlowInstance 状态变为 completed                │   │
│  │  可选：将表单数据持久化到业务数据表                               │   │
│  │  状态：✅ 流程引擎已支持                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 数据流

```
Editor 表单 Schema
       ↓
  FlowDefinition.formSchemaId + formPublishId
       ↓
  FlowInstance（发起时创建）
       ↓
  TaskInstance（审批时创建）
       ↓
  ApprovalLog（审批记录）
       ↓
  业务数据表（可选，流程结束后存储）
```

### 2.3 按钮事件系统

Editor 按钮支持以下事件类型：

| 事件类型 | 说明 | 参数 |
|---------|------|------|
| `startFlow` | 启动流程 | definitionId, variables |
| `endFlow` | 结束流程 | instanceId, reason |
| `api` | 自定义 API 调用 | method, url, params |
| `navigate` | 页面跳转 | url |
| `message` | 弹窗提示 | type, content |
| `dialog` | 打开弹框 | dialogId |

**按钮配置示例：**

```json
{
  "type": "button",
  "props": {
    "text": "提交审批",
    "type": "primary"
  },
  "events": [
    {
      "trigger": "click",
      "actions": [
        {
          "type": "startFlow",
          "definitionId": "{{$flow.definitionId}}",
          "variables": {
            "formData": "{{$form.values}}",
            "initiator": "{{$user.id}}"
          }
        },
        {
          "type": "message",
          "type": "success",
          "content": "流程已发起"
        },
        {
          "type": "navigate",
          "url": "/flow/instances/{{$response.instanceId}}"
        }
      ]
    }
  ]
}
```

### 2.4 审批表单模式

| 模式 | 说明 | 使用场景 |
|------|------|---------|
| `readonly` | 完全只读 | 审批人查看表单数据，不可编辑 |
| `editable` | 完全可编辑 | 发起人填写表单 |
| `partial` | 部分可编辑 | 审批人可填写审批意见，其他字段只读 |

**审批表单配置：**

```json
{
  "type": "userTask",
  "data": {
    "formMode": "partial",
    "editableFields": ["approvalComment", "approvalAction"],
    "readonlyFields": ["*"]
  }
}
```
