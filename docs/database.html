通用模式

项目中使用的通用数据建模模式。

🔑 UUID 主键

所有模型使用 UUID v4 作为 _id，而非 MongoDB 默认的 ObjectId。便于分布式生成和跨服务引用。

// Mongoose 定义 _id: { type: String, default: uuidv4 } // JSON 转换 ret.id = ret._id delete ret._id

⏰ 自动时间戳

所有模型启用 timestamps: true，自动维护 createdAt 和 updatedAt 字段。

// Schema 选项 { timestamps: true }

🔄 JSON 转换

统一的 toJSON 转换，移除 __v 和 _id，添加 id 字段。

// 密码字段额外移除 delete ret.password

索引策略

针对查询模式优化的索引设计。

模型 索引 类型 用途
FormSchema editId 唯一 编辑态快速查找
PublishedSchema sourceId 唯一 发布版本关联查询
PublishedSchema publishId 普通 通过 publishId 查询
User username 唯一 登录认证
User roles 普通 按角色查询用户
FlowDefinition name, status, createdBy 普通 流程列表筛选
FlowVersion definitionId + version 唯一 版本唯一性约束
FlowInstance definitionId 普通 按定义查询实例
FlowInstance status + updatedAt 复合 活跃实例排序
TaskInstance assignee + status 复合 用户待办任务
TaskInstance candidateUsers + status 复合 候选任务查询
TimerJob status + fireAt 复合 定时任务轮询

本地开发

使用 Docker 运行本地 MongoDB 实例。

🐳 Docker 命令

# 启动 MongoDB pnpm db:up # 停止 MongoDB pnpm db:down # 种子数据 pnpm db:seed

📦 Docker Compose

# docker-compose.yml services: mongodb: image: mongo:8 ports: - "27017:27017" environment: MONGO_INITDB_ROOT_USERNAME: formgrid MONGO_INITDB_ROOT_PASSWORD: formgrid MONGO_INITDB_DATABASE: formgrid
