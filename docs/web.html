核心能力

可视化设计器的核心功能模块。

🎨 可视化设计器

EditorView 三栏布局：左侧面板（部件库 + 结构树）+ 中间画布 + 右侧属性面板。工具栏居中放置面板抽屉控制、撤销/重做、复制/删除等操作按钮，右侧为缩放、模式切换、保存发布。

⚡ 事件引擎

纯逻辑层 eventEngine，通过 EventExecutionContext 注入运行时能力。支持 14 种动作类型：show/hide/open-dialog/close-dialog/switch-tab/set-value/submit/reset/emit/set-variable/trigger-event/post-message/close-tab/copy/refresh/api/navigate。

🔗 联动系统

6 种联动类型：visible / disabled / required / options / set-value / reset-fields。通过 watchFields 监听字段变化，条件表达式驱动 then/else 分支，支持 optionsApi 动态加载。

📦 拖拽系统

useDrag + useDragEditor 双层 composable 管理拖拽逻辑。dragStore 维护全局拖拽状态，支持从组件面板拖入画布、画布内排序、跨布局容器拖动，对齐辅助线（guidelines）和碰撞检测（collision）。

🔄 撤销 / 重做

editorStore 管理历史栈，主画布和弹窗编辑器独立管理。useSnapshot 提供快照恢复，useClipboard 处理剪贴板操作。路由守卫拦截未保存离开。

🌐 微前端集成

支持 qiankun 子应用模式（vite-plugin-qiankun）和 micro-app 模式（@micro-zoe/micro-app）。路由自动切换 memory history，AI 协作通过 @schema-form/socket 实现实时同步。

🔌 数据源配置

OptionsApiConfigDialog 支持 API 模式和字典码模式加载动态选项。responseNormalizer 标准化响应格式，optionsCache 缓存选项数据，requestQueue 管理请求队列。

📋 属性面板

Schema 驱动的右侧属性配置面板，通过 useRightPanelConfig 动态渲染。包含 15+ 专用编辑器：BorderEditor、SpacingEditor、ConditionBuilder、OptionsEditor、TableColumnsEditor、EventConfigDialog、VariableConfigDialog 等。

📐 自由布局

Widget 携带 position（x, y, w, h, zIndex）实现绝对定位自由布局。EditorCanvas 处理画布缩放、拖拽定位，EditorOverlay 提供选中框和拖拽手柄。支持画布尺寸预设切换。

📊 图表集成

9 种 ECharts 图表 Widget（Bar/Line/Pie/Scatter/Radar/Gauge/Heatmap/Funnel/Candlestick），通过 Schema 配置图表数据和样式，支持数据源绑定和动态刷新。

AI 集成

AI 微应用（packages/ai-app）通过 SSE 流式订阅 LangGraph 事件，实现对话式 Schema/Flow 生成。Editor 和 Flow 通过双通道接收 AI 生成物。

双入口模式

AiChatView（独立页面 /ai）

三栏布局：对话列表 + 聊天面板 + 预览面板。Agent 可切换（editor / flow / auto）。

AiSidebarView（抽屉模式）

400px 单栏，嵌入 Editor / Flow 宿主。Agent 固定，预览通过 bridge 推送到宿主。

useAiStore（共享状态）

Pinia store 管理：对话列表、当前消息、加载状态、生成物（currentSchema / currentFlow）、任务链。

sendMessage() 通过 fetch + ReadableStream 订阅 SSE 流，逐事件更新 messages 和生成物。

SSE 事件处理

9 种事件类型：

text tool_call schema flow task_chain agent_switch thinking tip done

text 事件累积到 content，tool_call 更新 toolCalls 数组，schema/flow 更新 currentSchema/currentFlow。

双通道写回

Socket 通道：通过 @schema-form/socket 的 emitAiApply() / emitAiPublished() 实时推送到宿主。

Bridge 通道：通过 postMessage（standalone）或 micro-app dispatch（qiankun）推送 ai:published / ai:open-in-editor 事件。

两个通道同时写入，确保无论宿主使用哪种嵌入方式都能收到生成物。

快速开始

# 安装依赖
pnpm install

# 启动开发服务器（端口 5173）
pnpm dev:editor

# 构建生产版本
pnpm build:editor

# 运行测试
pnpm --filter @schema-form/editor-web test

# 测试覆盖率
pnpm --filter @schema-form/editor-web test -- --coverage
