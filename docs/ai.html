LangGraph StateGraph

声明式图编排：5 个节点 + 条件边路由，工具调用循环由 LangGraph 自动管理

// graph/graph.ts — StateGraph 组装 import { StateGraph, END, START } from '@langchain/langgraph' import { ToolNode } from '@langchain/langgraph/prebuilt' // ToolNode 自动执行 LLM 返回的 tool_calls const editorToolNode = new ToolNode(editorTools) // 6 StructuredTool const flowToolNode = new ToolNode(flowTools) // 6 StructuredTool const builder = new StateGraph(AgentStateAnnotation) // 节点注册 .addNode('router', routerNode) .addNode('editor', editorAgentNode) .addNode('editorTools', editorToolNode) .addNode('flow', flowAgentNode) .addNode('flowTools', flowToolNode) // 入口 .addEdge(START, 'router') // Router → Agent（条件路由） .addConditionalEdges('router', routeAfterRouter) // Agent → ToolNode 或 END（检查是否有 tool_calls） .addConditionalEdges('editor', shouldCallTools) .addConditionalEdges('flow', shouldCallTools) // ToolNode → 回到对应 Agent（工具结果写回） .addEdge('editorTools', 'editor') .addEdge('flowTools', 'flow') const graph = builder.compile({ checkpointer })

图流转路径

START → router → editor / flow ↔ ToolNode → END

工具调用循环自动管理：Agent 返回 tool_calls 时路由到 ToolNode，执行后回到 Agent，无 tool_calls 时结束。

条件边函数

routeAfterRouter(state)

根据 state.currentAgent 分发到 editor / flow / END

shouldCallTools(state)

检查最后一条 AIMessage 是否有 tool_calls，有则路由到对应 ToolNode，否则 END

状态定义（Annotation.Root）

LangGraph Annotation 定义图状态，MessagesAnnotation 提供消息合并语义

// graph/state.ts — AgentStateAnnotation import { Annotation, MessagesAnnotation } from '@langchain/langgraph' export const AgentStateAnnotation = Annotation.Root({ // 消息字段继承 MessagesAnnotation（自动合并语义） ...MessagesAnnotation.spec, // 会话标识 sessionId: Annotation<string>({ reducer: (_, next) => next, default: () => '' }), conversationId: Annotation<string>({ reducer: (_, next) => next, default: () => '' }), // 当前活跃 Agent currentAgent: Annotation<'router' | 'editor' | 'flow'>({ reducer: (_, next) => next, default: () => 'router', }), // 业务上下文 context: Annotation<AIContext>({ reducer: (_, next) => next, default: () => ({ source: 'standalone', turnCount: 0 }), }), // 任务链（Router 分解复杂请求） taskChain: Annotation<TaskStep[]>({ reducer: (_, next) => next, default: () => [] }), currentStepIndex: Annotation<number>({ reducer: (_, next) => next, default: () => 0 }), // 工具状态、错误、澄清、偏好、历史摘要... needsTool: Annotation<boolean>({ ... }), toolResults: Annotation<AgentToolResult[]>({ ... }), error: Annotation<AIError | null>({ ... }), preferences: Annotation<Record<string, unknown>>({ ... }), historySummary: Annotation<string>({ ... }), })

智能体设计

三种专家 Agent 各司其职，Router 通过 LLM JSON Mode 分类意图，支持多步任务链

◉ Router Agent

职责：意图识别，将请求分发到对应专家

实现

ChatOpenAI (deepseek-chat) + JSON Mode，温度 0，maxTokens 200

输出

{ target: 'editor' | 'flow' | 'chain' }

支持 chain 模式拆分多步任务链

降级

LLM 失败时回退到关键词匹配（FLOW_KEYWORDS）

LLM 意图分类 JSON Mode 任务链

◉ Editor Agent

职责：生成/修改 Widget Schema

实现

ChatOpenAI (deepseek-chat) + JSON Mode + bindTools(editorTools)

温度 0.7，maxTokens 8192，streaming: true

领域知识

system prompt 由 buildEditorSystemPrompt(metadata) 动态生成

包含 Widget 目录、事件系统、联动系统、变量系统、输出格式

6 StructuredTool 动态 prompt JSON Mode

◉ Flow Agent

职责：生成/修改流程图（FlowGraph）

实现

ChatOpenAI (deepseek-chat) + JSON Mode + bindTools(flowTools)

温度 0.7，maxTokens 8192，streaming: true

领域知识

system prompt 由 buildFlowSystemPrompt(metadata) 动态生成

包含 12 种 BPMN 节点、校验规则、布局规则、典型模式

6 StructuredTool 动态 prompt JSON Mode

工具系统（14 StructuredTool）

所有工具使用 @langchain/core/tools 的 tool() 函数定义，zod schema 约束参数，返回统一的 ToolResult 格式

Editor Agent Tools（6）

Tool功能
search_schemas按关键词/类型搜索已有表单 Schema
get_schema_detail获取 Schema 完整 JSON 内容
search_published_schemas搜索已发布 Schema 版本
get_widget_catalogue获取 Widget 组件目录（动态 metadata）
semantic_search_schemas语义搜索 Schema（Jaccard 相似度）
validate_schema校验 Widget Schema 结构正确性

Flow Agent Tools（6）

Tool功能
search_flows搜索已有流程定义
get_flow_detail获取流程定义详情（含完整 FlowGraph）
search_users搜索用户（用于审批指派人配置）
search_schemas搜索表单（用于 userTask 表单绑定）
generate_schema调用 Editor Agent 生成表单 Schema
validate_flow校验 FlowGraph 结构正确性

Widget Tools（2）

Tool功能
query_widgets查询 Widget 组件目录（按分类筛选）
validate_widget_schema校验 Widget Schema 结构正确性

Widget Tools 为独立工具集，供 schemaGenerator 等辅助模块使用。

SSE 事件流

通过 graph.streamEvents() 获取 LangGraph v2 事件流，映射为 9 种 SSE 事件类型

type含义数据结构
textLLM 内容 token 流式输出{ content: string }
tool_call工具调用（calling / result 两阶段）{ phase, tools: [...] }
schema生成的 Widget Schema{ payload: Widget[], description: string }
flow生成的 FlowGraph{ payload: FlowGraph, description: string }
task_chain多步任务链步骤{ steps: TaskStep[], currentIndex }
agent_switchAgent 切换通知{ agent: 'editor' | 'flow' }
done流结束{ conversationId: string }
error错误信息{ content: string }

streamEvents 映射

LangGraph 事件映射为 SSE
on_chain_start (editor/flow)agent_switch
on_chain_end (router)task_chain（如果有）
on_chat_model_streamtext（跳过 router）
on_tool_starttool_call (phase: calling)
on_tool_endtool_call (phase: result) + schema/flow

@schema-form/shared-ai

构建时从 editor/flow 提取元数据，动态生成 system prompt，保证 AI 知识与编辑器/流程引擎单一数据源

metadata.json（构建时自动生成）

包含 Widget 目录（49 种）、Flow 节点目录（12 种）、系统类型（事件/联动/容器/变量）。

{ "version": "2026-06-02", "generatedAt": "2026-06-02T...", "widgets": [ { "type": "input", "group": "form", "displayName": "文本输入", "keyProps": ["placeholder", "maxlength"], "defaultSize": { "w": 280, "h": 44 }, "exposedValues": [...], "receivableEvents": [...] } ], "flowNodes": [...], "systems": { "eventActionTypes": [...], ... } }

promptBuilder.ts（动态 prompt 构建）

从 metadata 自动生成完整的 system prompt，包含 Widget 表格、Flow 节点表格、事件系统、联动系统、变量系统、输出格式、典型示例。

// 导出 3 个 prompt export function buildEditorSystemPrompt(metadata) // → Widget 表格 + 事件/联动/变量 + 输出格式 export function buildFlowSystemPrompt(metadata) // → BPMN 节点表格 + 校验规则 + 布局规则 export const ROUTER_SYSTEM_PROMPT // → 意图分类规则（稳定，不依赖 metadata）

前端 ai-app

独立 Vue 3 + Pinia 微应用，两种入口模式，通过 SSE 订阅 LangGraph 流式事件

两种入口模式

AiChatView（完整页面）

三栏布局：对话列表 + 聊天面板 + 预览面板

• 左侧：AiConversationList 对话历史列表
• 中间：AiChatPanel 消息流 + 输入框
• 右侧：AiPreviewPanel Schema/Flow 预览
• 入口：/ai 路由

AiSidebarView（抽屉模式）

400px 单栏，嵌入 Editor / Flow 宿主

• 仅 AiChatPanel 聊天面板
• 预览通过 bridge 推送到宿主
• Agent 固定显示为 Badge（不可切换）
• 宿主通过 micro-app 嵌入

useAiStore（Pinia）

共享状态管理：对话列表、当前消息、加载状态、生成物（currentSchema / currentFlow）、任务链。

sendMessage() 订阅 SSE 流，逐事件更新 messages 和生成物。

AiMessage.vue

消息渲染组件，支持 thinking 折叠区、toolCalls 折叠区（含参数/结果）、tip 提示、嵌入式 SchemaCard / FlowCard。

双通道写回

Socket：通过 @schema-form/socket 实时推送到宿主（emitAiApply / emitAiPublished）。

Bridge：通过 postMessage / micro-app 推送（ai:published / ai:open-in-editor）。

嵌入方式与通信协议

ai-app 通过 Socket + bridge 双通道与宿主通信，支持 micro-app 和 postMessage 两种模式

Editor 中嵌入

<!-- Editor 侧边栏 --> <micro-app name="ai-app" :url="AI_APP_URL" :data="{ source: 'editor', schemaId }" />

Flow 中嵌入

<!-- Flow 属性面板 --> <micro-app name="ai-app" :url="AI_APP_URL" :data="{ source: 'flow', flowId, nodeId }" />

通信协议（bridge.ts）

方向事件说明
ai-app → 宿主ai:preview-schema推送生成的 Schema 到编辑器预览
ai-app → 宿主ai:preview-flow推送生成的 FlowGraph 到流程设计器预览
ai-app → 宿主ai:published通知发布完成及绑定结果
ai-app → 宿主ai:open-in-editor请求宿主在编辑器中打开生成物
宿主 → ai-appai:set-context传入当前上下文（source, IDs）
宿主 → ai-appai:current-schema传入当前编辑器 Schema 供 AI 参考
