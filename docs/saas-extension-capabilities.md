# SaaS 扩展能力分析与集成方案

## 一、系统现状评估

| 维度 | 现状 | 成熟度 |
|---|---|---|
| **API** | RESTful CRUD（21 个路由），JWT 认证，Swagger 文档 | 中 |
| **多租户** | tenantPlugin（AsyncLocalStorage + Mongoose query hook），Tenant/Role/Permission 模型完整 | 中高 |
| **插件/Widget** | 49 个 Widget，registry.ts 静态注册（Map），编译时确定，无运行时扩展 | 低 |
| **流程引擎** | BPMN 设计器 + 实例引擎，内置节点类型，无自定义节点 | 中 |
| **AI 能力** | LangGraph agent + RAG + 20+ 工具，工具在 server 编译时确定 | 中 |
| **国际化** | 无 | 无 |
| **移动端** | 无 | 无 |
| **白标** | Config 模型支持 tenantId 隔离，可存品牌配置 | 低 |
| **导入导出** | Schema JSON 导入（含校验），Flow 导出 | 低 |
| **Webhook** | 无 | 无 |
| **第三方集成** | MicroApp 模型（微前端壳） | 低 |

## 二、扩展能力清单

### 1. 接入能力

| # | 能力 | 描述 | 现有基础 | 缺口 |
|---|---|---|---|---|
| 1.1 | **RESTful API 完善** | 所有资源完整 CRUD + 分页 + 过滤 | 已有 21 个路由 | 缺少 API Key 认证、速率限制、版本化 |
| 1.2 | **GraphQL 接入** | 提供 schema 查询语言 | 无 | 全新建设，优先级低 |
| 1.3 | **Webhook 回调** | 事件触发外部回调 | 无 | 需新建 WebhookModel + 事件总线 + 重试机制 |
| 1.4 | **钉钉/企微/飞书集成** | 消息推送、审批通知 | 流程通知已有基础 | 需抽象消息渠道层 |
| 1.5 | **数据导入导出** | Schema/Flow/表单数据导入导出 | Schema 导入已有 | 缺表单数据导出 |
| 1.6 | **API Key / OAuth2** | 外部系统认证 | 仅 JWT | 需新建 ApiKeyModel |

### 2. 插件能力

| # | 能力 | 描述 | 现有基础 | 缺口 |
|---|---|---|---|---|
| 2.1 | **自定义 Widget** | 用户注册新组件 | registry.ts 静态 Map | 无运行时注册机制 |
| 2.2 | **自定义流程节点** | 新增节点类型 | 固定节点类型 | 需节点注册表 + 执行器 |
| 2.3 | **自定义 AI 工具** | 用户定义 AI 工具 | 20+ 内置工具 | 无工具注册 API |
| 2.4 | **自定义主题/样式** | CSS 变量驱动主题切换 | CSS Module 隔离 | 需 design token 体系 |
| 2.5 | **自定义验证规则** | 用户定义校验逻辑 | 内置规则 | 需支持函数表达式 |
| 2.6 | **自定义数据源** | 注册外部 API | SchemaApiConfig | 缺管理面板 |

### 3. 扩展能力

| # | 能力 | 描述 | 现有基础 | 缺口 |
|---|---|---|---|---|
| 3.1 | **多租户隔离** | 数据/功能/配额隔离 | tenantPlugin 完整 | 缺配额执行 |
| 3.2 | **白标/定制化** | 品牌 logo/颜色/域名 | Config 模型 | 无前端主题消费 |
| 3.3 | **国际化 (i18n)** | 多语言 UI + 表单 | 无 | 需 vue-i18n |

| 3.5 | **权限精细化** | 字段级/操作级权限 | RBAC 完整 | 缺字段级权限 |
| 3.6 | **审计与合规** | 操作日志、变更追踪 | AuditLog 已有 | 缺变更对比 |

## 三、优先级排序（MoSCoW）

### Must Have（Q3 2026）

1. **Webhook 回调系统** -- SaaS 基础接入能力
2. **API Key 认证** -- 外部系统调用必须
3. **自定义数据源管理** -- 补上管理界面
4. **数据导出（表单提交）** -- 基本需求

### Should Have（Q4 2026）

5. **自定义 Widget 运行时注册** -- 为 Widget 市场打基础
6. **自定义流程节点** -- 扩展业务场景覆盖
7. **消息渠道抽象（钉钉/企微/飞书）** -- 对接企业 IM
8. **主题系统（Design Token）** -- 白标基础

### Could Have（2027 Q1）

9. **自定义 AI 工具注册** -- 用户定义 AI 工具
10. **国际化 (i18n)** -- 面向海外客户
11. **字段级权限** -- 高安全场景
12. **GraphQL** -- 锦上添花

### Won't Have（暂不考虑）

13. **移动端独立渲染器** -- 投入产出比低
14. **插件市场生态** -- 需要用户基数

## 五、关键决策点

1. **Widget 动态渲染方案选择**：建议先走预定义渲染器，后期开放模板编译
2. **流程节点执行器安全模型**：MVP 阶段只支持 HTTP 类型的自定义节点
3. **Webhook vs 事件总线**：统一为一个 EventBus，Webhook 是 EventBus 的一种消费者
4. **i18n 范围**：先做 UI 国际化，表单内容多语言作为后续需求
