## 五、持续 Loop 机制

### 5.1 Loop 流程

```
┌─────────────────────────────────────────────────────────┐
│                    回归检查 Loop                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 任务完成                                             │
│     ↓                                                    │
│  2. 运行相关模块测试（pnpm test）                         │
│     ↓                                                    │
│  3. 测试全部通过？                                        │
│     ├── 是 → 更新本文档状态 → 进入下一个任务              │
│     └── 否 → 分析失败原因 → 分级（P0/P1/P2）             │
│              ↓                                           │
│         4. P0？                                          │
│           ├── 是 → 立即修复 → 回到步骤 2                 │
│           └── 否 → 记录问题 → 继续当前任务                │
│                    ↓                                     │
│               5. 每周全量回归                             │
│                 ├── 更新可用性评分                        │
│                 ├── 更新问题清单                          │
│                 └── 调整下个 Sprint 优先级                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.2 自动化检查命令

```bash
# 全量测试（每周回归）
pnpm test

# 单模块测试（任务完成后）
pnpm --filter @schema-form/editor-web test
pnpm --filter @schema-form/flow-shared test
pnpm --filter @schema-form/flow-web test
pnpm --filter @schema-form/server test
pnpm --filter @schema-form/ai-app test
pnpm --filter @schema-form/portal test

# 构建检查（部署前）
pnpm build
```
