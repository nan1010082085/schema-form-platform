# 全局总览

本项目重构目标：彻底解决旧项目硬编码样式、样式冲突、组件嵌套混乱、渲染错乱、模式逻辑不统一、版本混乱问题。

所有规则为**最终定稿，不可新增、不可修改、不可自定义实现**。

# 1. 全局核心架构规范

1. 编辑器 / 预览 / 运行时 **共用同一套渲染容器、同一套组件、同一套 Schema**。

2. 组件层级铁律：

    - 基础组件、业务组件 **不互嵌、不组合、不嵌套**

    - 所有组件**只能放入布局组件内**

3. 全量开启 CSS Module，组件样式完全隔离。

4. 全量删除历史硬编码样式，所有样式 Schema 驱动。

5. 架构分层：Store 管状态、Composables 管逻辑、Components 管 UI。

# 2. 三大运行模式完整规范

## 2.1 编辑模式

1. 所有组件**禁用一切交互**（点击、输入、弹窗、联动全部失效）。

2. 仅允许：拖拽、选中、增删、属性修改、层级调整。

3. 组件拖拽画布**生成永久唯一ID，终身不变**。

4. 编辑历史最多保存 **10 条快照**，自动淘汰旧记录。

5. 数据保存至【编辑表】（草稿版）。

## 2.2 预览模式

1. 全真模拟发布环境，所有交互、联动、弹窗全部生效。

2. **不生成 VID、不写入发布表、不持久化任何发布数据**。

3. 沿用编辑阶段组件ID。

## 2.3 发布模式（二选一）

### 可交互模式

- 页面全部表单、按钮、联动、弹窗、操作功能全部正常可用、可见。

### 无交互只读模式（严格执行）

- 所有表单输入框禁用

- **所有表单内部按钮隐藏**

- **所有工具栏、工具组件按钮正常显示、可用**

- **文件列表、文件下载功能永久保留可用**

# 3. 版本双表存储规范（最终）

## 编辑表

- 存储页面草稿、最新页面 JSON、最多10条编辑历史快照。

## 发布表

- 每次发布生成全新页面 VID

- **新发布直接覆盖旧发布数据**

- 只保留**最新一份发布版本**，无历史版本留存

- 线上运行时只读发布表最新版本

# 4. 组件 ID 全局规则

1. 组件ID结构：`组件Key + 5位随机Hash`

2. 生成时机：拖拽入画布瞬间生成，永久不变

3. 弹窗独立全局唯一ID

4. 页面维度使用 VID 作为发布版本唯一标识

# 5. 组件样式规范（强制重构）

1. 清除全部：固定表格样式、固定宽高、固定边框、全局硬编码样式

2. 所有组件样式由 Schema 驱动，支持可视化配置

3. 所有组件开启 CSS Module 隔离

## 5.1 布局组件默认规范

- 默认高度：44px

- 行布局：100% 自适应撑满

- 列布局：支持栅格配置

- 四边边框**独立可控（显隐/颜色/粗细/样式）**

- 仅布局组件支持 Label

## 5.2 表单组件默认规范

- 默认无边框

- 默认宽100%、高44px

- 自带 Label 配置

# 6. 渲染加载机制规范

1. 单组件固定加载顺序：  

组件加载 → 拉取配置 → 拉取业务数据 → 动态渲染

1. 全局**串行队列加载渲染**，杜绝渲染错乱

2. 所有组件独立渲染、独立请求、互不依赖，仅挂靠布局组件

# 7. 编辑器完整 UI 布局规范

## 7.1 顶部工具栏

撤销、重做、保存草稿、预览、发布、缩略图显隐

- **画布禁止缩放，仅缩略图支持显示/隐藏**

## 7.2 左侧抽屉（双Tab）

Tab1：组件库（布局/基础/业务组件分类）  

Tab2：页面目录树

- 树形展示真实布局嵌套结构

- 支持组件重命名、定位、删除、上下移动、复制

## 7.3 中间画布

- 固定尺寸、不可缩放

- 严格遵循：组件只放布局内、组件互不嵌套

## 7.4 缩略图

实时预览画布结构，仅可控显隐

## 7.5 右侧属性面板（完整5大Tab+底部功能）

1. 基础配置：实例名、唯一ID（只读）、显隐、默认值、占位、提示

2. 样式配置：宽高、边距、对齐、背景、圆角、阴影、四边独立边框

3. 数据配置：接口、请求方式、字段映射、静态数据、默认数据

4. 事件&联动：组件事件、触发条件、目标组件、执行动作

5. 高级配置：权限、只读模式适配、白名单配置

6. 底部：JSON预览、JSON一键复制、恢复默认Schema

# 8. 弹窗编辑规则

1. 弹窗内部复用主画布渲染、拖拽体系

2. 弹窗内组件同样遵循：**只嵌套布局、不互嵌**

3. 弹窗独立唯一ID、独立DOM挂载

# 9. 项目目录&架构强制统一

## stores（只存状态）

- pageStore：页面结构、组件树、布局结构、编辑状态、UI抽屉状态

- renderStore：渲染队列、加载状态

- publishStore：VID、发布模式、交互权限状态

## composables（全部业务逻辑）

### core 底层核心

useConstant、useIdGenerate、useStyleHandle、usePageStorage、useRequestQueue

### business 编辑器业务

useModeControl、useInteractionControl、useDragEditor、useEditorLayout、useLeftPanelManage、useRightPanelConfig

## schema

defaultSchema.ts 统一存放所有组件默认样式结构

## components

- editor：编辑器UI组件

- layout：布局组件（唯一可嵌套容器）

- base：基础组件（独立、CSS Module、无互嵌）

- business：业务组件（独立、CSS Module、无互嵌）

# 10. 标准开发重构顺序

1. 全局开启 CSS Module

2. 搭建标准化目录、Store、Composables 空架构

5. 实现ID生成、串行渲染、双表存储、历史快照

6. 实现三模式切换与严格交互权限控制

7. 完整开发编辑器左右面板、属性面板、联动配置

8. 弹窗拖拽体系开发

9. 全量联调、样式统一、交互闭环、发布闭环
