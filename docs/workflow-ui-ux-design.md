# Workflow 系统 UI/UX 完整设计方案

## 一、设计原则

基于现有项目的设计令牌体系（`packages/shared/styles/tokens.css` + `_variables.scss`），确保与 Editor 和 Flow 两个包的视觉语言完全统一。

**核心参考**：n8n 的节点面板分组 + Dify 的变量选择器交互。目标用户是非技术人员，所有配置必须通过 UI 完成。

## 二、设计规范

### 2.1 颜色系统（复用现有令牌 + Workflow 专用扩展）

| 节点类型 | 色值 | CSS 变量 |
|----------|------|----------|
| 开始节点 | #26A036 | --color-success |
| 结束节点 | #E50113 | --color-danger |
| Editor 节点 | #0060A2 | --color-primary |
| Flow 节点 | #7b1fa2 | --node-accent-flow |
| AI 节点 | #F09700 | --color-warning |

| 通知节点 | #e65100 | --node-accent-notify |

### 2.2 节点尺寸

- 标准节点：180px x 80px
- 条件节点：160px x 80px（菱形）

## 三、页面布局

### 3.1 Workflow 列表页

- 卡片/列表视图切换
- 搜索和筛选
- 状态标签（草稿/已发布/已归档）
- 卡片宽度：minmax(280px, 1fr)，3 列网格

### 3.2 Workflow 画布页

- 左面板：260px（节点面板，可折叠）
- 画布：flex: 1（Vue Flow）
- 右面板：320px（属性面板，可折叠）
- 工具栏：40px
- 状态栏：32px

### 3.3 节点组件结构

```
┌─────────────────────────────┐
│ [图标] 节点名称        [状态]│  ← 头部
├─────────────────────────────┤
│  摘要信息（1-2 行）          │  ← 内容区
├─────────────────────────────┤
│ 输入连接点        输出连接点 │  ← 连接点
└─────────────────────────────┘
```

### 3.4 节点类型

1. **开始节点**：绿色圆形，直径 40px
2. **结束节点**：红色圆形，直径 40px
3. **Editor 节点**：蓝色左边框 4px，表单/页面配置
4. **Flow 节点**：紫色左边框 4px，流程配置
5. **AI 节点**：橙色左边框 4px，AI 任务配置
6. **条件节点**：黄色菱形，条件判断
7. **通知节点**：橙色左边框 4px，通知配置

## 四、交互规范

### 4.1 拖拽交互

- 从左面板拖入画布：半透明预览 → 放置高亮 → 创建节点
- 画布内拖拽：节点跟随 → 对齐辅助线 → 推入历史栈
- 连线交互：临时连线 → 连接点高亮 → 创建连线

### 4.2 快捷键

| 操作 | 快捷键 |
|------|--------|
| 保存 | Ctrl+S |
| 撤销 | Ctrl+Z |
| 重做 | Ctrl+Y |
| 删除 | Delete |

### 4.3 响应式策略

| 断点 | 左面板 | 画布 | 右面板 |
|------|--------|------|--------|
| >= 1440px | 260px | flex: 1 | 320px |
| 1024-1439px | 48px 图标栏 | flex: 1 | 320px |
| < 1024px | 折叠 | flex: 1 | 底部抽屉 |
